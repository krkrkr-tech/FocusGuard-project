# Repository Self-Evaluation (FocusGuard)

I would rate my repository around **6/10**.

Overall, the project works and the main idea is clear — it’s a browser extension that helps users stay focused by blocking distracting websites. However, I can see that it still looks more like a learning project than a fully polished portfolio project.

---

## README

I think my README is **okay but not strong enough**.
It explains what the project does and lists features, but it is missing important parts like:

* how to install the project
* how to use it step by step
* what technologies I used

Because of that, someone new might not immediately understand how to run or test the project.

---

## Folder Structure

The structure is simple and works, but **everything is mostly in the root folder**, which is not ideal.

For a bigger or more professional project, I should organize it better, for example:

* separate `src`, `assets`, etc.
* group related files together

Right now, it looks a bit basic.

---

## File Naming

File names are generally clear (like `popup.js`, `background.js`, etc.), so it’s easy to understand what each file does.

But I think I don’t follow a strict naming convention everywhere, so it could be more consistent.

---

## Essential Files

Some important files are present:

* README
* LICENSE
* manifest.json

But I’m missing things like:

* `.gitignore`
* possibly dependency/config files

This makes the project look less complete from a professional perspective.

---

## Commit History

This is probably one of my weaker areas.

My commits are not very descriptive (for example: “update”, “fix”), and I don’t follow a structured format like:

* `feat:`
* `fix:`
* `refactor:`

Because of that, it’s harder to understand the development process.

---

## Final Thoughts

I think the project is:

* good as a practice or study project
* but not yet strong enough for a portfolio

To improve it, I need to:

* improve the README
* organize the structure better
* use better commit messages
* add missing configuration files
